Welcome to ServiceMeow

What is this?

This is a small proof-of-concept application intended to demonstrate how an enterprise can provide self-service SAML app registration to its developer teams

Why make this?

You could spend your time manually registering SAML applications, but imagine if you didn't have to. Consider the following alternatives that you could spend your time on:

  • Focus on resolving compliance issues or MRAs
  • Instead of just maintaining your service, you now have time to actually improve it to better enable developers/customers
  • Learn a new skill
  • Drink beer all day

How does it work?

It makes some API calls to the Azure graph API. Special thanks to this guide: https://docs.microsoft.com/en-us/graph/application-saml-sso-configure-api

It really isn't special. The secret sauce is/will be in the specific security controls required at MUFG

See GitHub

Tech Stack

  • Alpine Linux Docker
  • Node.js 14.x
  • Koa web framework
  • Federated sign-on enabled with Azure AD and Open ID Connect
  • Hosting Provider (TBD)
