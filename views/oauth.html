OAuth/OIDC

Not available in this POC

Enabling self-service for OpenID Connect or OAuth applications would probably be even easier. In my experience, there are far less variances in an OIDC configuration than SAML configurations. For instance, the set of claims that OIDC relying parties expect tend to be inline with the standard while many SAML apps look for small differences.

OpenID Connect and OAuth also provide solutions where SAML may not. Single page apps, devices without keyboards etc... The protocol tends to be developer friendly as well. I was able to turn it on for this portal using a few lines of code having never used Azure AD before. By extension, self-service OIDC might better serve developers and future technology decisions.
